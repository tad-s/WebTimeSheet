作成日: 2026-04-05

## 1. システム概要

社員の勤怠（出退勤）および交通費を管理するWebアプリケーション。

| 項目 | 内容 |
|------|------|
| 言語 | PHP 8.2 |
| フレームワーク | なし（素のPHP） |
| UI | Bootstrap 4系 |
| DB | MySQL 5.7（さくらインターネット） |
| DBホスト | mysql624.db.sakura.ne.jp |
| DB名 | yokumono49_works |

## 4. 機能仕様

### 4-1. 認証

#### ログイン
- 社員番号（3桁の数字）とパスワードで認証
- 社員番号入力: 3桁数字 → DBの`user_no`は「e」+3桁形式（例: `074` → `e074`）
- パスワード: bcryptで検証（`password_verify`）
- 認証成功でセッション（`$_SESSION['USER']`）にユーザー情報を格納
- CSRF対策: トークンによる二重送信防止

#### ログアウト
- セッション破棄

### 4-2. 勤怠入力（index.php）

#### 画面表示
- 当月の日付一覧をカレンダー形式で表示
- 土日・祝日は行の背景色で区別（holiday-row / saturday-row / sunday-row）
- 祝日判定: 固定祝日、ハッピーマンデー、春分・秋分、振替休日、国民の休日に対応
- なりすまし中は対象ユーザー名をヘッダーに表示

#### 勤怠登録・編集
- モーダルダイアログで日付ごとに入力
- **社内勤務**: 勤務区分、出勤時刻、退勤時刻、休憩時間、備考
- **現場勤務**: 勤務区分、出勤時刻、退勤時刻、休憩時間、備考
- 時刻丸め処理:
  - 出勤時刻 → **15分単位切り上げ**
  - 退勤時刻 → **15分単位切り捨て**
- 欠勤（5）・公休（8）: 時刻を空白、備考入力必須
- 帰社日（9）: 退勤時刻を17:30に自動設定
- 退勤が出勤より前の場合はエラー（24時以降は翌日扱いで計算）

#### 編集権限

| ユーザー種別 | 編集可能期間 |
|-------------|-------------|
| 管理者（auth_type=1） | 制限なし |
| edit_flg=1のユーザー | 制限なし |
| 一般ユーザー（auth_type=0） | 当月 + 先月（当月10日まで） |

#### CSVエクスポート
- `export_csv.php`: 社内勤務データ（`社員番号_氏名_勤務表(社内用)_yyyy-mm.csv`）
- `export_csv_field.php`: 現場勤務データ

#### 一括アップロード（bulk_upload.php）
- CSVファイルで複数日分の勤怠を一括登録
- 時刻丸め処理あり

### 4-3. 交通費入力（fare.php）

- モーダルダイアログで入力
- 入力項目: 経費種別、移動日、出発地、到着地、金額（片道）、往復チェック、備考
- 往復の場合: total_cost = cost × 2
- CSVエクスポート:
  - `fare_export_csv.php`: 自分の交通費
  - `fare_export_csv_field.php`: 現場交通費
- 編集権限: 勤怠と同じルール適用

### 4-4. 管理者機能（admin/）

管理者（auth_type=1）のみアクセス可能。

#### 社員一覧（user_list.php）
- 管理者のteam_noに基づいて表示範囲を制限
  - team_no=0（スーパー管理者）: 全社員表示
  - team_no≠0: 同チームのみ表示

#### 社員登録（user_register.php）
- 社員番号（3桁数字）、氏名、パスワード、権限を登録
- 社員番号はDB保存時に「e」プレフィックスを付与
- パスワード: 4文字以上50文字以内、bcryptでハッシュ化

#### 社員編集（user_edit.php）
- 氏名、パスワード、権限、チームNO、edit_flg、在籍フラグ等を編集

#### なりすまし機能（impersonate.php）
- 管理者が特定社員として代理で勤怠入力できる機能
- セッションに`IS_IMPERSONATING`フラグと元管理者情報（`ORIGINAL_ADMIN`）を保持
- `return_from_impersonation.php`で管理者セッションに戻る

#### 交通費管理（fare_view.php）
- 特定社員の交通費データを参照・登録可能
- CSVエクスポート可能

## 6. 権限体系

```
スーパー管理者（auth_type=1, team_no=0）
  └─ 全社員の勤怠・交通費を参照・管理可能

チーム管理者（auth_type=1, team_no≠0）
  └─ 同チームの社員のみ管理可能

一般ユーザー（auth_type=0）
  ├─ edit_flg=1: 過去月も編集可能
  └─ edit_flg=0: 当月 + 先月10日まで編集可能
```

## 7. セキュリティ

- パスワード: `password_hash`（bcrypt）でハッシュ化
- CSRF対策: トークンによる二重送信防止（`set_token` / `check_token`）
- XSS対策: 出力時に`htmlspecialchars`（`h()`関数）でエスケープ
- SQLインジェクション対策: PDOのプリペアドステートメント使用
- セッション管理: PHP標準セッション

## 10. 特記事項

1. **社員番号の二重管理**: `work`テーブルに`user_no`カラムが冗長保持されている（本来は`user_id`だけで十分）
2. **fare.user_idがNULL**: 交通費テーブルの`user_id`がNULLのレコードが多数存在し、`user_no`で紐付けている
3. **24時以降対応**: 退勤時刻が翌日になる場合（例: 25:00など）を考慮した時刻計算実装済み
4. **モーダル自動表示**: コード上でコメントアウトされており、エラー時のみモーダル表示
5. **帰社日（work_type=9）**: 退勤時刻を17:30に自動固定する特別ルール
